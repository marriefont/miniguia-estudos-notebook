# miniguia-estudos-notebook
Atividade realizada no bootcamp da DIO.

Contexto e Objetivos
O assunto central deste caderno é o domínio de fundamentos técnicos essenciais para um Consultor de Dados. Em um cenário de megadados, é vital entender como transformar dados brutos em decisões acionáveis.

Objetivos de Estudo:

Compreender a distinção entre métodos científicos, técnicas e algoritmos.

Dominar o uso de bibliotecas fundamentais em Python (NumPy, Pandas, Matplotlib).

Diferenciar o tratamento de dados estruturados (tabelas) e não estruturados (imagens/séries temporais).

Aplicar IA (NotebookLM) como ferramenta de síntese e organização de conhecimento complexo.

Curadoria de Fontes
Para este estudo, foram selecionadas e processadas as seguintes fontes de referência:

Estatística e Ciência de Dados (Morettin & Singer, 2021) - Link para o PDF.

O que é Data Analytics? (AWS) - Guia fundamental sobre a aplicação prática de análise de dados no mercado.

Materiais de Prática: Introdução a estruturas de repetição, Pandas e visualização de dados.

-------------------------------------------------------------------------------

Engenharia de Prompts e "Cicatrizes" (Processo de IA)
Nesta seção, documento como interagi com a IA para extrair a melhor síntese dos dados.

Variações de Prompts Testadas:
Prompt 1 (Geral): "Resuma os arquivos." -> Resultado: Muito superficial.

Prompt 2 (Estruturado): "Atue como um Professor de Ciência de Dados. Com base nas fontes, crie uma comparação técnica entre dados estruturados e não estruturados." -> Resultado: Ótima distinção técnica.

Prompt 3 (Refinamento): "Explique a diferença entre método e algoritmo usando o exemplo de estourar pipoca mencionado na aula." -> Resultado: Gerou o insight perfeito para o glossário.

Dificuldades e Troubleshooting:
Dificuldade: A IA inicialmente confundiu a sintaxe de C com a de Python ao ler as gravações das aulas.

Solução: Ajustei o prompt para especificar: "Considere apenas as boas práticas da guia PEP 8 do Python mencionada na fonte."

Cicatriz de Aprendizado: Percebi que o raciocínio por trás da escolha do algoritmo é tão importante quanto o código em si.

Análise Exploratória (EDA): A importância de consultar o pesquisador original antes de excluir outliers para não enviesar o estudo.

📝 Glossário
Algoritmo: Conjunto de passos finitos (ex: a receita para estourar pipoca).

Método/Técnica: Procedimento sistemático geral (ex: aquecer o milho no fogão ou micro-ondas).

Dados Estruturados: Dados organizados em instâncias e atributos (tabelas).

Interpretação vs Compilação: Diferença fundamental entre a execução do Python e da linguagem C.
